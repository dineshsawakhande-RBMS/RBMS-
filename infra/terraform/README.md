# RBMS Infrastructure (Terraform)

Provisions the AWS footprint for the Retail Business Management System:

- **networking** — VPC, 2 public + 2 private subnets across 2 AZs, IGW, per-AZ
  NAT gateways, route tables, and security groups (ALB → ECS → RDS).
- **secrets** — Secrets Manager entries for DB credentials and the JWT signing
  key (passwords generated by `random_password`).
- **rds** — Multi-AZ PostgreSQL 16 with subnet/parameter groups, enforced SSL,
  enhanced monitoring, automated backups, and deletion protection.
- **ecs** — ECR repo, Fargate cluster + service + task definition, ALB with
  HTTP→HTTPS redirect, target group health checks, CPU/memory autoscaling, and
  a CloudWatch log group.
- **s3_cloudfront** — Private, versioned, encrypted buckets for product images,
  employee docs, and business docs; CloudFront distribution with Origin Access
  Control in front of the images bucket.

## Prerequisites

- Terraform >= 1.6
- AWS CLI configured with credentials that can create the above resources
- An ACM certificate (optional) in the deployment region for HTTPS on the ALB

## Remote state bootstrap (one-time)

```bash
aws s3api create-bucket \
  --bucket rbms-terraform-state-<account-id> \
  --region ap-south-1 \
  --create-bucket-configuration LocationConstraint=ap-south-1
aws s3api put-bucket-versioning \
  --bucket rbms-terraform-state-<account-id> \
  --versioning-configuration Status=Enabled

aws dynamodb create-table \
  --table-name rbms-terraform-locks \
  --attribute-definitions AttributeName=LockID,AttributeType=S \
  --key-schema AttributeName=LockID,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST \
  --region ap-south-1
```

Then uncomment the `backend "s3"` block in `versions.tf` and run `terraform init`.

## Apply

```bash
cp terraform.tfvars.example terraform.tfvars   # edit values
terraform init
terraform plan  -out tfplan
terraform apply tfplan
```

### Apply order / dependencies

The root module wires everything in one graph; Terraform resolves ordering
automatically (networking → secrets → rds → ecs, s3_cloudfront in parallel).
The ECS service is created with a placeholder image when `api_image` is empty
so `terraform apply` succeeds before the first ECR push. CI/CD then deploys the
real image via `force-new-deployment` (the service ignores `task_definition`
changes so Terraform and the pipeline don't fight).

## Useful outputs

| Output                   | Use                                          |
| ------------------------ | -------------------------------------------- |
| `alb_dns_name`           | Point DNS / test the API                     |
| `ecs_cluster_name`       | `--cluster` for deploy workflow              |
| `ecs_service_name`       | `--service` for deploy workflow              |
| `rds_endpoint`           | Connection string host                       |
| `db_secret_arn`          | Migrations / app secret reference            |
| `cloudfront_domain_name` | CDN domain for product images                |

## Notes

- `random_password` and RDS `password` are stored in Terraform state — keep the
  state bucket private and encrypted (it is, via the backend config).
- Provider alias `aws.us_east_1` is passed to the CDN module for any future
  CloudFront ACM certificate (CloudFront certs must live in us-east-1).
- Set `acm_certificate_arn` to enable the HTTPS listener and HTTP→HTTPS redirect.
